# Acknowledgements

## AI-Assisted Development

This project was built through a collaborative process between a human
contributor and an AI coding assistant ([Kiro](https://kiro.dev)).

### How it was built

All source code, tests, documentation, configuration, and CI/CD pipelines
in this repository were generated by Kiro in response to iterative,
human-provided requirements. The development process followed a
conversational, incremental model:

1. The human contributor described a feature or requirement in natural language.
2. Kiro generated the corresponding code, tests, and documentation.
3. The human reviewed the output, ran the tests, and provided follow-up
   direction to refine, extend, or correct the work.
4. This cycle repeated across the full lifecycle of the project.

### Human contributions

- Defined the overall project concept and feature set
- Provided all requirements, constraints, and acceptance criteria
- Reviewed every code change before it was accepted
- Guided architectural decisions (one class per file, OOP structure,
  RFC 865 compliance, OWASP threat modelling, Cloudflare Worker target)
- Validated test results and approved the final state of the project

### AI contributions

- Generated all Python source files (`src/`, `worker/src/`)
- Wrote all unit, integration, benchmark, and security tests (`tests/`, `worker/tests/`)
- Authored Sphinx documentation (`docs/`)
- Produced the OWASP threat model (`docs/threat_model.md`)
- Created CI/CD pipeline configuration (`.github/workflows/ci.yml`)
- Wrote all supporting files (`Makefile`, `README.md`, `CHANGELOG.md`,
  `CONTRIBUTING.md`, `SECURITY.md`, `requirements*.txt`, `setup.py`)

### Transparency statement

The use of AI assistance is disclosed in the spirit of transparency and
to accurately represent how modern software development can be conducted.
The human contributor takes full responsibility for the content,
correctness, and suitability of this project.

AI-generated code carries the same obligations as human-written code:
it must be reviewed, tested, and understood by the person who ships it.
