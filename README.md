# Greeting

> **🌐 Live demo:** [greeting-worker.cancun.workers.dev](https://greeting-worker.cancun.workers.dev/) — a new inspirational quote every minute, colour-coded to your local time of day.

A time-aware, colourised terminal greeting application written in Python.

Prints a greeting based on the time of day, with the message colour changing to reflect the current period:

| Period    | Hours         | Colour   |
|-----------|---------------|----------|
| Morning   | 06:00 – 11:59 | Yellow   |
| Afternoon | 12:00 – 17:59 | Cyan     |
| Evening   | 18:00 – 20:59 | Magenta  |
| Night     | 21:00 – 05:59 | Blue     |

## Requirements

- Python 3.10+
- [colorama](https://pypi.org/project/colorama/)

## Installation

```bash
pip install -r requirements.txt
```

## Usage

```bash
python src/greeting.py
# Good morning, World!  [09:30 AM]
# The only way to do great work is to love what you do.
# — Steve Jobs
```

Custom name without quote:

```python
from greeting import Greeting
Greeting("Alice").run()
```

With Quote of the Day (RFC 865):

```python
from greeting import Greeting
from quote_provider import QuoteProvider

Greeting("Alice", quote_provider=QuoteProvider()).run()
```

Run the RFC 865 TCP server (requires port > 1023 for unprivileged use):

```python
import threading
from quote_service import QuoteService

service = QuoteService(port=1717)
t = threading.Thread(target=service.start_tcp, daemon=True)
t.start()
# connect with: nc 127.0.0.1 1717
```

## Project Structure

```
src/
  time_of_day.py     # TimeOfDay enum — period classification, colours, salutations
  greeting.py        # Greeting class — builds and prints the colourised message
  quote_provider.py  # QuoteProvider — RFC 865 compliant quote corpus + shuffle-deck selection
  quote_service.py   # QuoteService  — RFC 865 TCP/UDP server (port 17)
  telemetry.py       # TelemetryStore — async SQLite telemetry (nanosecond precision)
  stats.py           # StatsReporter — min/max/mean/p95/p99 from telemetry DB
worker/
  src/entry.py       # Cloudflare Worker entry point (HTTP API)
  src/local_server.py # Local dev server (stdlib http.server, no Node needed)
  src/cf_telemetry.py # D1-backed async telemetry adapter
  src/time_of_day.py  # colorama-free TimeOfDay (CSS hex colours)
  src/quote_provider.py # Quote corpus (copied from src/)
  schema.sql         # D1 migration schema
tests/
  test_time_of_day.py    # Unit tests for TimeOfDay
  test_greeting.py       # Unit tests for Greeting
  test_quote_provider.py # Unit tests for QuoteProvider
  test_quote_service.py  # Unit + integration tests for QuoteService (real sockets)
  test_telemetry.py      # Unit tests for TelemetryStore and StatsReporter
  test_security.py       # Security regression tests (T-01 through T-22)
  test_integration.py    # End-to-end integration tests
  test_benchmark.py      # Benchmark tests
worker/tests/
  test_worker.py         # Worker routing + telemetry tests (54 tests)
scripts/
  dev.sh             # Convenience script wrapping all Makefile commands
docs/                # Sphinx documentation source
Makefile             # Build targets: run, test, worker-dev, worker-test, docs, audit, reset, report
```

## Key Commands

```bash
make run NAME="Alice"          # greet Alice with quote + telemetry
make run NAME="Alice" RESET=1  # wipe DB first, then run
make reset                     # wipe telemetry database
make report                    # print all-time stats
make test                      # 167 CLI tests
make worker-dev                # local Worker HTTP server on :8787
make worker-test               # 54 worker tests
make audit                     # CVE scan
make docs                      # build Sphinx HTML docs

# Or via the convenience script:
sh scripts/dev.sh help
NAME=Alice sh scripts/dev.sh run
```

## CI/CD

GitHub Actions runs on every push and pull request to `main`:
- Runs the full test suite across Python 3.10, 3.11, 3.12, and 3.13
- Scans dependencies for CVEs with `pip-audit`
- Builds Sphinx documentation and uploads it as an artifact
